# Contexte du projet

WoodyCraft est une entreprise qui vend des puzzles en ligne via une plateforme web. Avec l’augmentation du nombre de produits et de commandes, la gestion administrative devient plus complexe.

Afin de faciliter cette gestion, une **application mobile destinée à l’administrateur** doit être développée. Cette application permettra de gérer les commandes, le catalogue de produits et les stocks de manière simple et rapide.

Elle utilisera la **même base de données que l’application web** et communiquera avec celle-ci via **une API REST**.

---

# Objectifs de l’application

L’application WoodyCraft Admin a pour objectif de **simplifier la gestion quotidienne de la boutique** pour l’administrateur.

Elle permettra de :

- **Gérer les commandes** : consulter les commandes, voir leur détail, les valider ou les marquer comme expédiées.
- **Gérer le catalogue** : ajouter, modifier ou supprimer des puzzles dans le catalogue.
- **Gérer les stocks** : consulter les quantités disponibles et ajuster les niveaux de stock.
- **Consulter un tableau de bord** : avoir une vue rapide sur les commandes et l’état des stocks.

---

# Problématiques auxquelles l’application répond

Cette application répond à plusieurs besoins :

- **Centraliser la gestion** des commandes, produits et stocks.
- **Gagner du temps** dans les tâches administratives.
- **Améliorer le suivi des commandes et des stocks**.
- **Permettre une gestion plus rapide et mobile** pour l’administrateur.
